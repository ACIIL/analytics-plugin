What is Kundi Bora?

Kundi Bora is an analyitcs plugin for the Ushahidi platform 2.7.2. It takes the data from an existing Ushahidi install and provides the ability to build charts on the fly. It is a plugin, so installation is as easy as dropping the analytics folder into the plugins directory of an existing Ushahidi install.

Ushahidi?

Ushahidi is an open source software project that first debut during the 2008 kenyan elections in which it was used to track unrest that hit the nation during this time. The public would send in reports of violence or vote tampering and they would be displayed on a map of the region. Incidents that occurred close together were clustered more or less depending on the level of detail in the map.

Since its debut, Ushahidi has continued to grow. As a platform for crowd sourcing information on a specific event or situation, it has been used to track unrest for other elections but also for more benign issues such as the incidence of potholes in a city.

OK, But Why Do I Need Analytics?

You might not. If all your interested in is how many incidents there are in your city or country, then analytics probably won't do much for you. However, if you need to tease out trends from a data set then Kundi Bora is here to help.

Charts, you say? What kind of charts?

As it stands now, Kundi Bora serves up charts primarily with the flotcharts javascript library and includes a proof of concept for integrating the D3 javascript visualization library into the plugin which opens up the door a myriad of data visualization options.

Why Can't I just install these libraries myself?

You can! However, unless your comfortable querying the database, rolling what you get into a JSON object that the libraries can use then writing the code that will actually use that data to create the different chart type, then I would stick to Kundi Bora. Lots of work has gone into not only creating the charts you see when you install the plugin but also writing the code that allows making more charts much easier for the uninitiated. This includes functions that allow you to get the data you want with out having to know any SQL commands. It also includes code that simplifies working with the libraries by removing the need to write the chart code yourself. Simply call a chart function with the data you want and the chart is yours. If you don't want to touch any code, we've included filtering with a pie or line chart that allows you to select how the data (you can filter by category, country, date, and keyword) is displayed. We are working to bring this feature to all chart types in future releases.

The bottom line is, if you need analytics now for you deployment and don't have time to learn the nitty gritty of the Kohana platform Ushahidi uses or how to write the code that will produce the charts then take advantage of the work we've done. Why reinvent the wheel when someone is offering you one for free?
